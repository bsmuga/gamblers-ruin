# agent-configuration

## Purpose

Defines how the project's Claude Code subagents are structured and kept consistent —
which agents exist, where their definitions live, and how per-agent memory relates to
version control.

## ADDED Requirements

### Requirement: Agent roster is defined by committed definitions

Every project subagent SHALL be defined by exactly one Markdown file in
`.claude/agents/<name>.md`, where `<name>` matches the `name` field in the file's
frontmatter. The roster SHALL include the `ocaml-implementer` agent. There SHALL be no
agent that the project relies on without a corresponding definition file.

#### Scenario: ocaml-implementer is present

- **WHEN** the contents of `.claude/agents/` are listed
- **THEN** a file `ocaml-implementer.md` exists whose frontmatter `name` is
  `ocaml-implementer`

#### Scenario: Every definition is a real agent

- **WHEN** any `.claude/agents/<name>.md` file exists
- **THEN** its frontmatter `name` equals `<name>`

### Requirement: Per-agent memory is runtime state, not source

Per-agent memory under `.claude/agent-memory/` SHALL be treated as runtime state that is
regenerated by the harness on demand. It SHALL NOT be committed to version control:
`.gitignore` SHALL ignore `.claude/agent-memory/`. The `memory: project` setting on agent
definitions SHALL be retained so the memory feature stays available.

#### Scenario: Memory directory is git-ignored

- **WHEN** `git check-ignore .claude/agent-memory/<any-agent>/` is run
- **THEN** the path is reported as ignored

#### Scenario: Memory feature stays enabled

- **WHEN** an agent definition in `.claude/agents/` is read
- **THEN** its frontmatter still declares `memory: project`

### Requirement: No empty placeholder directories are left as trash

The repository SHALL NOT carry or leave on disk empty per-agent memory directories as
untracked placeholders. Empty `.claude/agent-memory/<name>/` directories SHALL be removed;
the harness recreates a directory only when an agent actually writes a memory.

#### Scenario: No empty memory directories remain on disk

- **WHEN** the working tree is inspected after this change
- **THEN** no empty `.claude/agent-memory/<name>/` directory exists that has never held a
  memory file

#### Scenario: Memory directory regenerates on first write

- **WHEN** an agent with `memory: project` writes its first memory after the directories
  were removed
- **THEN** the harness recreates the appropriate `.claude/agent-memory/<name>/` directory,
  and it remains git-ignored
