## Context

The project configures five Claude Code subagents under `.claude/agents/`
(`math-concept-docs`, `ocaml-implementer`, `probability-math-reviewer`,
`senior-code-reviewer`, `test-author`), each with `memory: project` frontmatter. That
setting causes the harness to create a per-agent directory under `.claude/agent-memory/`.
Those directories are currently empty. Git does not track empty directories, so they
appear in neither the index nor `git status` — they exist only on disk as orphaned
placeholders. The `.claude/agents/` tree itself is still untracked and will be committed
as part of normal work.

## Goals / Non-Goals

**Goals:**

- A complete, self-consistent agent roster where every committed definition is a real
  agent and `ocaml-implementer` is included.
- A clear, version-control-correct treatment of per-agent memory: keep the feature, never
  commit its storage.
- No empty placeholder directories left on disk or implied by the repo structure.

**Non-Goals:**

- Changing what any agent does (prompts, models, tools) beyond ensuring the roster is
  complete.
- Building tooling to validate the agent/memory invariant automatically; this change
  establishes the convention, not an enforcement mechanism.
- Touching OCaml source, Dune config, or the build.

## Decisions

**Treat `.claude/agent-memory/` as runtime state and git-ignore it.**
Memory content is generated by agents at runtime and is machine-/session-specific, not
authored source. Committing it (or its empty skeleton) adds churn and noise. Adding a
single `.gitignore` entry keeps the working tree clean regardless of when the harness
recreates directories.
*Alternative considered — commit the structure with `.gitkeep` files:* rejected because it
makes empty folders permanent fixtures and invites committing real memory later, which is
session-specific trash.
*Alternative considered — drop `memory: project` entirely:* rejected because the agents
were deliberately configured with memory; removing the feature is a behavior change, not
hygiene.

**Delete the existing empty memory directories now.**
They carry no information and git cannot commit them anyway. The harness recreates the
relevant directory the first time an agent writes a memory, so deletion is non-destructive.

**Keep the roster as the five existing definitions.**
`ocaml-implementer.md` and `test-author.md` already exist alongside the three reviewer/doc
agents, so the roster is already complete; this change confirms and documents that rather
than adding files.

## Risks / Trade-offs

- [A future contributor adds a `.gitkeep` back, re-committing empty dirs] → The `.gitignore`
  entry plus the spec requirement document the intended convention, making such a change
  easy to spot and reject in review.
- [Git-ignoring the directory could hide a memory file someone wanted to share] → Acceptable:
  per-agent memory is explicitly defined as non-shared runtime state; deliberate sharing
  would be a separate, intentional decision.

## Migration Plan

1. Add `.claude/agent-memory/` to `.gitignore`.
2. Remove the empty `.claude/agent-memory/*` directories from the working tree.
3. Commit the agent definitions and the `.gitignore` change together.

Rollback is trivial: revert the `.gitignore` entry; deleted directories held no data.
